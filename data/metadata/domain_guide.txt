[서강대학교 학사 DB 도메인 가이드]
이 DB는 서강대학교 학사 데이터베이스입니다.
수록 데이터 범위: 2025년도 (1학기, 2학기)

==================================================
## 1. "학점" 의도 구분 규칙
==================================================
"학점"이라는 단어는 문맥에 따라 의미가 다릅니다. 반드시 아래 기준으로 구분하세요.

[성적 학점 → takes.grade 기반 GPA 계산]
- 키워드: "평균 학점", "GPA", "성적", "학점이 몇이야", "학점 몇점"
- 예시: "컴퓨터공학과 학생들 평균 학점이 몇이야?" → takes.grade로 GPA 계산
- 테이블: takes, student (필요시 course, department JOIN)

[이수 학점 수 → course.credits 합산]
- 키워드: "몇 학점 들어야", "이수 학점", "취득 학점", "학점 수", "졸업 학점"
- 예시: "졸업하려면 몇 학점 들어야 해?" → takes JOIN course WHERE grade IS NOT NULL 집계
- 테이블: takes, course
- 주의: student.tot_cred는 신뢰할 수 없습니다. 반드시 takes JOIN course로 직접 계산하세요.

## 3. 질문 유형별 필요 테이블 힌트
==================================================
학생 기본 정보 조회          → student
학과 정보 조회               → department
성적/수강 내역               → takes, course, section
교수 정보                    → instructor
강의 담당 교수               → teaches, instructor, course
지도교수                     → advisor, instructor, student
강의실/수업 시간             → section, classroom, time_slot
선수과목                     → prereq, course
학과별 통계 (평균 GPA 등)    → student, takes, course, department

==================================================
## 4. 학기/연도 주의사항
==================================================
- semester 값: '1학기' 또는 '2학기' (Fall/Spring 등 영문 사용 금지)
- 수록 데이터 범위: 2025년도만 존재
- "이번 학기", "최근 학기" → year = 2025로 처리
- "현재 수강 중" → takes.grade IS NULL
- "수강 완료" / "이수한" → takes.grade IS NOT NULL
